OOP Principles

Object Oriented Programming

An approach to designing and building applications that are flexible, natural, well-crafted, and testable by focusing on objects that interact efficiently with one another. It is a widely used programming concept that reduces development times—making it easier to read, reuse, and maintain your code.

There are four major principles that helps with clean object-oriented coding – Encapsulation, Abstractions, Inheritance and Polymorphism

Encapsulation

It is a way to hide the information (or encapsulate) of the data and functions implementation details within a class, it means that when such object is instantiated that it will contain its properties and methods in that object

Example: In the Calculator program, the class Calculation is an example of encapsulation as it encompasses all the data in the object; the instance properties (starting with “def __init__”) whenever calculation is instantiated these are the properties that will be independent, and this is called encapsulation.

Abstraction

Abstraction in Object-Oriented Programming helps to reduce the complexity of the design and implementation process of software. It is the process of hiding the real implementation of an application from the user and showing only essential features. It is achieved by using abstract classes and methods. In a way an extension to encapsulation feature as Abstraction hides the implementation details.

Example: In the calculator.py the class calculation is the blueprint for other class, it allows to create set of methods within any child classes built from the abstract (base) class.

Polymorphism

Polymorphism defines the ability to take different forms. Meaning a single method can behave different depending on the type of object that calls it. It allows us to define methods in the child class with the same name as defined in their parent class.

Example: In the Calculator program the built-in function len() calculates the length of an object depending upon its type. If an object is a string, it returns the count of characters, and if an object is a list, it returns the count of items in a list. In the calculator this was used to count the history of calculation.

Inheritance

A relationship between classes whereby child (or derived) classes inherit all the members of the parent (or base) class. Inheritance is a powerful feature in object-oriented programming as It refers to defining a new class with little or no modification to an existing class.

Example: In the calculator project – addition, subtraction, multiplication, and division are the child classes, and the calculation class is the parent class (Base class) as it provides functionality to child classes.
