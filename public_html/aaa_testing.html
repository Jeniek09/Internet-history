AAA Testing

Importance of Testing

Testing provides a practical feedback loop for development and shows how individual components of an application affects the results. Therefore, to conduct a thorough test of code we should follow the AAA testing pattern.

AAA Testing

Arrange-Act-Assert pattern is a constructive way to structure test cases to test the calculator code.

  • Arrange: it is the first step to start the testing process.

  • Act: The desired method which will be tested in this testing.

  • Assert: is the expected outcome for the conducted test above in the act step.

AAA Testing Steps

  1. For the calculator program as seen in the image above, the first part of starting the testing is by instantiating the class (calc) in the code.

  2. Followed by the main method/function that is to be tested to see if the calculator is interpreting the code correctly. In this case it was the addition method that is called to be tested in the Act step.

  3. And lastly to verify whether the expected answer/result is met, you insert the “assert” code and specify the anticipated answer. For the example mentioned, when adding value_a = 4 to base value 0 the expected answer should be 4.

  4. This AAA test was further tested with multiple values and for different operations such as multiplication, division, and subtraction.
