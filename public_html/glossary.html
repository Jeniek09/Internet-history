Glossary - Object Oriented (OO) Terminology/ Definitions

OOP Terminology:

  • Instantiation: Instantiating a class is creating a copy of the class which inherits all class variables and methods.

  • Object: An object (instance) is an instantiation of a class. When class is defined, only the description for the object is defined. Therefore, no memory or storage is allocated.

  • Attributes: also known as variables inside the class so that all objects created from that class would have these attributes by default.

  • Class: can be considered as the blueprint or a bag for the object. It contains all the details about the methods and functionality so that every object created can access the class.

  • Base class: Is the parent of the child classes created in the python.

  • Derived class: Child class is the class that inherits from another class, and it also called derived class.

  • Methods: are functions defined inside the body of a class. They are used to define the behaviors of an object.

  • Static method: property or method that is not bound to instance, it is used to perform utility tasks in isolation. They have limited use because they don’t have access to the attributes of an object. Static method is defined by using the @staticmethod decorator or the staticmethod() function to define a static method.

  • Instance method: The instance method acts on an object’s attributes; it is bound to the object of a class. It can modify the object state by changing the value of instance variables because it can access and modify both class and instance variable.

  • Class method: Used to access or modify the class state. It can modify the class state by changing the value of a class variable that would apply across all the class objects.

  • Namespace: A namespace is basically a system to make sure that all the names in a program are unique and can be used without any conflict. When a class definition is entered, a new namespace is created, and used as the local scope for all assignments to local variables go into this new namespace.

  • Constructor: Constructors are generally used for instantiating an object. The task of constructors is to initialize (assign values) to the members of the class when an object of the class is created. In Python the __init__() method is called the constructor and is always called when an object is created.
